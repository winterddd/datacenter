2.  Shared Hardware Queue Technologies: AIOE & GIPC Proposals
    
    1.  Ren Guo from Alibaba DAMO Academy presented the AIOE (Atomic I/O Enqueue) and GIPC (Global Inter-Process Communication) proposals, along with corresponding reference model implementations (including QEMU and test cases).
        

Details:

In the era of artificial intelligence, a single CPU architecture can no longer meet the demands of diverse intelligent computing workloads. Consequently, heterogeneous computing has emerged as the mainstream approach. Integrating different domain-specific accelerators (DSAs) into computing systems enhances overall computational efficiency. For instance, high-dimensional tensor computing tasks can be offloaded to TPUs/NPUs/GPGPUs, while data stream processing tasks are delegated to DPUs. Efficient management of DSAs in heterogeneous systems has become a prominent industry focus, driving several technological advancements. PCIe 5.0 and CXL 2.0 have introduced Deferrable Memory Write (DWMr) TLP.

*   Armv8.7/9.2 has incorporated ST64B instructions for 64-byte atomic /O enqueue operations
    
*   The x86 architecture has implemented ENQCMD instructions with comparable functionality
    

To help RISC-V adapt to this trend, the presentation introduced the Atomic I/O Enqueue Extension (AIOE) and its usage with virtualization. The AIOE extension is designed for RV64 ISA, which includes one PMA definition, two U-mode instructions, two S-mode instructions, a single S-mode CSR, and two envcfg control bits. The presentation also introduced how to use AIOE under virtualization scenarios, which involves the new proposal for RISC-V IOMMU: G-stage table in Process Context (GIPC). With the help of AIOE and GIPC, RISC-V could explore new heterogeneous programming paradigms from HPC to embedded scenarios.
