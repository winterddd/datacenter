# Minutes

1.  QoS with Proposal for RISC-V in Cloud, Presenter: Jay Chen (Alibaba)
    
    Jay Chen, a software engineer from Alibaba Cloud, presented architecture of RISC-V QoS and ARM MPAM, and a comparison between  RISC-V QoS and ARM MPAM. Jay mainly introduced the current design and implementation details of RISC-V QoS, with a focus on the hardware architecture, software architecture, and fundamental capabilities of ARM's MPAM. It covered common MPAM features such as cache and memory bandwidth monitoring, as well as isolation capabilities for cache and memory bandwidth. By combining practical applications in current cloud scenarios, He provided usage examples. Finally, He compared the differences and similarities between RISC-V's CBQRI and ARM's MPAM. Subsequently he proposed a priority-based QoS scheme, introducing the idea of dynamically configuring priorities.
    
    Jay and Vedvyas discussed the differences between ARM and RISC-V architectures, particularly focusing on cache partitioning and monitoring. Vedvyas explained that in RISC-V, the CBQRI interface generalizes capacity and bandwidth control to various components, not just caches, and can be applied to resources like TLBs and fabric bandwidth. Guo clarified that while the current proposal focuses on cache partitioning, the architecture allows for more flexible implementation of capacity blocks, which could be implemented as cache ways or other units depending on the specific design.Jay subsequently proposed a priority-based QoS scheme, introducing the idea of dynamically configuring priorities.Vedvyas explained that while priority could be implemented through configuration registers, there were concerns about making it static rather than dynamic, particularly given that different request types.
    
2.  Applications and Challenges of Trusted Platfrom Modules, Presenter: Liqiang Ni (ZTE)
    

Liqiang Ni, a software engineer from ZTE Corporation, presented on the application and challenges of trusted platform modules (TPM) in the RISC-V ecosystem. He explained that TPM provides security functions like key generation, platform authentication, and protection against attacks, and is widely adopted in X86 and ARM architectures. Ni highlighted that while TPM integration offers advantages such as robust security and broad platform support, there are challenges with TCG SAPI inspection and standardized initialization methods for RISC-V platforms. He concluded by seeking recommendations from the RISC-V community on how to address these interoperability issues and drive forward the development of TPM solutions for RISC-V platforms.
